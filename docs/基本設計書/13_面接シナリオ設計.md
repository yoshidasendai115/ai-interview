本ドキュメントは、外国人採用面接のAI面接官（HeyGen）用シナリオ設計を定義する。外国人採用に関する各種サイト情報を総合的に分析し、求められる人材像を明確化した上で、面接シナリオ・質問バンクの構造を定義する。

## 13.3 面接シナリオ構造

### 13.3.1 シナリオフロー（時間軸ベース・45分想定）

```
【導入】5分
├── 挨拶・アイスブレイク
├── 面接の流れ説明
└── リラックス促進

【過去】10分（経験・背景の確認）
├── 来日理由・経緯
├── 職歴・学歴
├── 日本語学習の経緯
└── 困難を乗り越えた経験

【現在】10分（能力・適性の確認）
├── 現在の日本語学習状況
├── 強み・弱み
├── 仕事に対する価値観
└── 希望条件の確認

【未来】10分（意欲・ビジョンの確認）
├── 志望動機
├── キャリアプラン
├── 日本での生活設計
└── 会社への貢献イメージ

【条件確認】5分
├── 勤務条件の確認（残業・シフト）
├── 在留資格の確認
└── 家族の同意確認

【クロージング】5分
├── 逆質問
├── 今後の流れ説明
└── 挨拶
```

### 13.3.2 質問カテゴリ構成

| カテゴリID | カテゴリ名 | 時間軸 | 所要時間 | 質問数 |
|-----------|----------|--------|---------|--------|
| introduction | 導入・アイスブレイク | - | 5分 | 5問 |
| past_experience | 過去（経験・背景） | past | 10分 | 16問 |
| present_ability | 現在（能力・適性） | present | 10分 | 16問 |
| future_vision | 未来（意欲・ビジョン） | future | 10分 | 12問 |
| conditions | 条件確認 | - | 5分 | 6問 |
| closing | クロージング | - | 5分 | 5問 |

### 13.3.3 JLPTレベル別対応

| レベル | 質問の特徴 | 簡易版使用 | フォローアップ深度 |
|-------|----------|-----------|------------------|
| N1 | 謙譲語・尊敬語を含むビジネス敬語 | No | 深い（3回まで） |
| N2 | 標準的なビジネス敬語 | No | 標準（2回まで） |
| N3 | 丁寧語（です・ます調） | No | 標準（2回まで） |
| N4 | 簡易な丁寧語、基本漢字のみ | Yes | 浅い（1回まで） |
| N5 | 最も簡易な表現、ひらがな多め | Yes | 浅い（1回まで） |

> **HeyGen話速設定について**
> ※ HeyGen話速設定（JLPT_VOICE_RATE）は 12_面接フロー制御 12.2 を参照

### 13.3.4 JLPTレベル別質問の実装

POC面接セッションでは、質問バンク（60問）からJLPTレベルに応じた質問を動的に取得する。

#### API設計

```
GET /api/v1/questions?jlptLevel=N3
```

> **レスポンス形式**: 10_API仕様 10.5 を参照

#### 質問選択ロジック

| 項目 | 仕様 |
|------|------|
| **質問数** | 10問 / セッション |
| **選択基準** | カテゴリ別に均等選択（導入2問 + 過去2問 + 現在2問 + 未来2問 + クロージング2問） |
| **テキスト選択** | N1-N3: `question_ja` / N4-N5: `question_simplified` |
| **発話テキスト** | 全レベル: `question_reading` |

#### レベル別質問スタイル

| レベル | テキスト | 例 |
|--------|----------|-----|
| **N1-N3** | `question_ja`（標準） | 「本日はお越しいただきありがとうございます」 |
| **N4-N5** | `question_simplified`（簡易版） | 「今日は来てくれてありがとう」 |

### 13.4.3 動的プレースホルダー置換

質問テンプレート内の「〇〇」プレースホルダーは、面接開始時に業界に応じた都道府県名に動的に置換される。

![プレースホルダー置換フロー](./images/設計書_13.4_プレースホルダー置換フロー.png)

#### 対象質問

| ID | テンプレート | 置換後の例 |
|----|-------------|-----------|
| Q42 | 勤務地は〇〇ですが、通勤は大丈夫ですか？ | 勤務地は愛知県ですが、通勤は大丈夫ですか？ |

#### 置換ロジック

1. 質問テンプレートから「〇〇」プレースホルダーを検出
2. 選択された業界IDに紐づく都道府県リストをマスターデータから取得
3. リストからランダムに1件を選択
4. プレースホルダーを選択された都道府県名で置換

> マスターデータに定義された都道府県からのみ選択されるため、不正確な地名が生成されることはない。

## 13.5 評価基準（採用適性評価）

> **注意**: 本セクションは「**採用適性評価**」の評価基準を定義する。
> 日本語能力評価（語彙・文法・内容・敬語、0-100点スケール）については07_評価ロジックを参照。

### 13.5.1 5段階評価ルーブリック（採用適性評価）

| スコア | ラベル | 説明 |
|--------|-------|------|
| 5 | 優秀 | 期待を大きく上回る。即戦力として活躍できる。 |
| 4 | 良好 | 期待を上回る。適切なサポートで成長が見込める。 |
| 3 | 標準 | 期待通り。一般的な水準を満たしている。 |
| 2 | 要改善 | やや期待を下回る。追加確認や研修が必要。 |
| 1 | 不十分 | 期待を大きく下回る。採用は慎重に検討。 |

### 13.5.2 リスク検出・ブロック機能（採用リスク検出）

本システムでは、面接回答とAI分析を通じて採用リスクを自動検出し、Red Flags（即不合格）とWarning Signs（要注意）として分類・通知する。

![採用リスク検出フロー](./images/設計書_13.5_採用リスク検出フロー.png)

### 13.5.3 Red Flags（即不合格レベル）

| 検出項目 | 検出方法 | 関連質問 | アクション |
|---------|---------|---------|----------|
| 在留資格に問題がある | 事前情報確認 + Q45回答分析 | Q45 | 即座に不合格フラグ設定 |
| 特定技能試験に合格していない | 事前情報確認 + Q45回答分析 | Q45 | 即座に不合格フラグ設定 |
| 職歴に3ヶ月以上の説明できない空白がある | Q07,Q11回答のAI分析 | Q07,Q11 | 即座に不合格フラグ設定 |
| 面接中に虚偽の発言があった | 回答間の整合性AI分析 | 全質問 | 即座に不合格フラグ設定 |

**Red Flag検出時のシステム動作:**
- 採用推奨: 「不採用を推奨」
- レポート: Red Flag詳細を明記
- mintoku連携: 即時アラート送信

### 13.5.4 Warning Signs（要注意レベル）

| 検出項目 | 検出方法 | 関連質問 | アクション |
|---------|---------|---------|----------|
| 家族の同意が得られていない | Q44回答分析 | Q44 | Warning表示、追加確認推奨 |
| 短期間での退職歴が多い | Q07,Q52回答分析 | Q07,Q52 | Warning表示、追加確認推奨 |
| 給料だけが志望動機 | Q30,Q35,Q36回答分析 | Q30,Q35,Q36 | Warning表示、追加確認推奨 |
| 質問への回答が極端に短い・長い | 回答時間・文字数分析 | 全質問 | Warning表示 |
| 日本語能力が申告より明らかに低い | スコア乖離検出（7.9参照） | Q20,Q24等 | Warning表示、レベル再評価推奨 |

**Warning Sign検出時のシステム動作:**
- 採用推奨: 「慎重に検討」
- レポート: 注意事項として記載
- 追加確認: 推奨質問を提示

### 13.6.2 テンプレートデータ構造

```typescript
interface ScenarioTemplate {
  id: string;                    // テンプレートID
  name: string;                  // 日本語名称
  name_en: string;               // 英語名称
  description: string;           // テンプレートの説明
  duration_minutes: number;      // 総所要時間（分）
  jlpt_levels: JLPTLevel[];      // 対応JLPTレベル
  industries: string[];          // 対応業界（"all"で全業界）
  flow: PhaseDefinition[];       // フェーズ定義の配列
}

interface PhaseDefinition {
  phase: PhaseType;              // フェーズ種別
  duration_minutes: number;      // フェーズの所要時間
  question_ids: string[];        // 使用する質問ID
  notes: string;                 // 面接官向けの注意事項
  additional_questions?: string[]; // 業界特化の追加質問（任意）
}

type PhaseType =
  | 'introduction'      // 導入・アイスブレイク
  | 'past_experience'   // 過去（経験・背景）
  | 'present_ability'   // 現在（能力・適性）
  | 'future_vision'     // 未来（志望動機・ビジョン）
  | 'conditions'        // 条件確認
  | 'closing';          // クロージング
```

### 13.6.3 標準面接（45分）フロー詳細

外国人採用の標準的な面接フロー。過去・現在・未来の時間軸に沿って質問を行う。

| フェーズ | 時間 | 質問ID | 目的・注意事項 |
|---------|------|--------|---------------|
| 導入 | 5分 | Q01, Q02, Q03 | リラックスした雰囲気を作る。緊張をほぐす。 |
| 過去 | 10分 | Q06, Q07, Q08, Q09, Q10 | 来日理由と職歴を中心に確認。困難への対処能力を見る。 |
| 現在 | 10分 | Q18, Q19, Q22, Q23, Q24 | 強み・弱みと日本語学習意欲を確認。 |
| 未来 | 10分 | Q30, Q31, Q32, Q33, Q39 | 志望動機とキャリアビジョンを確認。定着意向を見極める。 |
| 条件 | 5分 | Q40, Q43, Q44, Q45 | 勤務条件と資格要件を確認。 |
| クロージング | 5分 | Q46, Q48, Q50 | 逆質問を受け、今後の流れを説明。 |

### 13.6.6 JLPTレベル別適応設定

シナリオテンプレートはJLPTレベルに応じて適応される。

| JLPTレベル | 話速 | 簡略化 | 深掘り回数 | 特徴 |
|-----------|------|--------|----------|------|
| N1 | 通常 | なし | 3回 | ビジネス日本語使用。敬語も積極的に使う。論理的説明を求める。 |
| N2 | 通常 | なし | 2回 | 標準的なビジネス日本語。丁寧語中心。理解確認を時々挟む。 |
| N3 | ゆっくり | なし | 2回 | やや簡潔な表現。平易な敬語。必要に応じて言い換え。 |
| N4 | かなりゆっくり | あり | 1回 | 簡単な語彙、短文、やさしい日本語。ジェスチャーも活用。 |
| N5 | かなりゆっくり | あり | 1回 | 最も簡単な表現。基本的な語彙のみ使用。簡易版質問を使用。 |

> **※ HeyGen話速設定との対応は 12_面接フロー制御 12.2 を参照**
